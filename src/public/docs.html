Cursor To OpenAI API 文档

欢迎使用 Cursor To OpenAI API！我们提供了与 OpenAI 完全兼容的 API 接口， 让您可以轻松地将 Cursor 的 AI 模型集成到您的应用程序中。

OpenAI 兼容

完全兼容 OpenAI API 格式，无需修改现有代码

多种模型

支持 Claude、Cursor 等多种先进的 AI 模型

流式响应

支持实时流式响应，提供更好的用户体验

安全可靠

企业级安全保障，稳定的服务可用性

身份验证

Cursor To OpenAI API 使用 API 密钥进行身份验证。您需要在每个请求的 Authorization 标头中包含您的 API 密钥。

授权标头格式 
Authorization: Bearer YOUR_API_KEY

获取 API 密钥

请前往 管理页面 添加和管理您的 API 密钥。每个 API 密钥都可以配置多个 Cursor Cookie。

快速开始

以下是一个简单的示例，展示如何使用我们的 API 进行聊天补全：

cURL 示例 
curl -X POST "http://localhost:3010/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "claude-3-sonnet",
    "messages": [
      {
        "role": "user",
        "content": "你好，世界！"
      }
    ],
    "max_tokens": 100,
    "temperature": 0.7
  }'
Python 示例 
import requests

url = "http://localhost:3010/v1/chat/completions"
headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
}

data = {
    "model": "claude-3-sonnet",
    "messages": [
        {
            "role": "user",
            "content": "你好，世界！"
        }
    ],
    "max_tokens": 100,
    "temperature": 0.7
}

response = requests.post(url, headers=headers, json=data)
result = response.json()
print(result["choices"][0]["message"]["content"])
JavaScript 示例 
const response = await fetch('http://localhost:3010/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    model: 'claude-3-sonnet',
    messages: [
      {
        role: 'user',
        content: '你好，世界！'
      }
    ],
    max_tokens: 100,
    temperature: 0.7
  })
});

const result = await response.json();
console.log(result.choices[0].message.content);

聊天补全

创建一个聊天补全响应。这是主要的 API 端点，用于与模型进行对话。

POST /v1/chat/completions

请求参数

参数 类型 必需 描述
model string 是 要使用的模型 ID
messages array 是 对话消息列表
max_tokens integer 否 生成的最大 token 数量
temperature number 否 采样温度，0-2 之间，默认 0.7
stream boolean 否 是否启用流式响应，默认 false
stop string/array 否 停止生成的字符序列

消息格式

每个消息对象应包含以下字段：

消息对象 
{
  "role": "user",  // "system", "user", "assistant"
  "content": "Hello, how are you?"
}

请求示例

请求体 
{
  "model": "claude-3-sonnet",
  "messages": [
    {
      "role": "system",
      "content": "你是一个有帮助的AI助手。"
    },
    {
      "role": "user",
      "content": "请解释什么是机器学习。"
    }
  ],
  "max_tokens": 500,
  "temperature": 0.7,
  "stream": false
}

响应示例

200 OK 
{
  "id": "chatcmpl-7QyqpwdfhqwajicIEznoc6Q47XAyW",
  "object": "chat.completion",
  "created": 1677652288,
  "model": "claude-3-sonnet",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "机器学习是人工智能的一个子领域..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 56,
    "completion_tokens": 31,
    "total_tokens": 87
  }
}

流式响应

当设置 stream: true 时，API 将返回流式响应，允许您实时接收生成的内容。

流式响应示例 
data: {"id":"chatcmpl-123","object":"chat.completion.chunk","created":1694268190,"model":"claude-3-sonnet","choices":[{"index":0,"delta":{"role":"assistant","content":""},"finish_reason":null}]}

data: {"id":"chatcmpl-123","object":"chat.completion.chunk","created":1694268190,"model":"claude-3-sonnet","choices":[{"index":0,"delta":{"content":"Hello"},"finish_reason":null}]}

data: {"id":"chatcmpl-123","object":"chat.completion.chunk","created":1694268190,"model":"claude-3-sonnet","choices":[{"index":0,"delta":{"content":"!"},"finish_reason":null}]}

data: {"id":"chatcmpl-123","object":"chat.completion.chunk","created":1694268190,"model":"claude-3-sonnet","choices":[{"index":0,"delta":{},"finish_reason":"stop"}]}

data: [DONE]

处理流式响应

JavaScript 流式处理 
const response = await fetch('/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    model: 'claude-3-sonnet',
    messages: [{ role: 'user', content: 'Hello!' }],
    stream: true
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split('\n');
  
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = line.slice(6);
      if (data === '[DONE]') return;
      
      try {
        const parsed = JSON.parse(data);
        const content = parsed.choices[0]?.delta?.content;
        if (content) {
          console.log(content);
        }
      } catch (e) {
        // 忽略解析错误
      }
    }
  }
}

错误处理

API 使用标准的 HTTP 状态码来表示成功或失败。所有错误响应都包含一个错误对象。

错误响应格式

错误响应 
{
  "error": {
    "message": "Invalid API key provided",
    "type": "invalid_request_error",
    "code": "invalid_api_key"
  }
}

HTTP 状态码

200

OK

请求成功

400

Bad Request

请求格式不正确或缺少必需参数

401

Unauthorized

API 密钥无效或缺失

429

Too Many Requests

请求频率超出限制

500

Internal Server Error

服务器内部错误

请求限制

为了确保服务的稳定性和公平使用，我们实施了以下请求限制：

每分钟请求数

60 次

每个 API 密钥每分钟最多 60 次请求

每小时请求数

3,600 次

每个 API 密钥每小时最多 3,600 次请求

并发请求数

10 个

每个 API 密钥最多 10 个并发请求

超出限制时

当您超出请求限制时，API 将返回 HTTP 429 状态码。建议实施指数退避策略来处理限制。

最佳实践

API 密钥安全

  • 将 API 密钥存储在环境变量中
  • 不要在前端代码中暴露密钥
  • 定期轮换 API 密钥
  • 为不同用途创建单独的密钥

性能优化

  • 使用流式响应提升用户体验
  • 适当设置 max_tokens 限制
  • 根据用途选择合适的模型
  • 实施请求缓存策略

错误处理

  • 始终检查 HTTP 状态码
  • 实施重试机制和退避策略
  • 妥善处理网络超时
  • 记录错误信息用于调试
